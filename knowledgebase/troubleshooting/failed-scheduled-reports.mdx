This guide explains how to troubleshoot scheduled reports that are failing to run or appearing blank. Follow these steps before escalating the issue.

## Diagnostic steps

<Steps>
<Step title="Check the report">
Go to the report in question and run it to see if it generates data successfully. If it runs with data, the report's parameters and variables may have been changed by the user. Since "my reports" are user-specific, ask the client for screenshots to identify the changes.
</Step>
<Step title="Review the report query">
If the customer provides screenshots showing additional input in the parameters, check the report queries on their domain. Unlike "my reports", report queries are the same for all users on a single domain.

Once you locate the report query, look through the script. If a field (e.g. "delivery address name") has been added to the user's report but is missing from the report query, you will need to add it.

The path format is: `node.original_delivery_address.name` or `node.goods_out_pick_list.id`
</Step>
<Step title="Check Mailchimp (if the report ran successfully)">
If the report successfully ran, check whether it was actually sent. Navigate to: **Mailchimp > Automations > Transactional Email > Launch App > Outbound** and enter the email address.

- If the email status is **Rejected**, it was likely filtered as spam or blacklisted. Advise the client to contact their IT department.
- If the email status is **Delivered**, the email was sent without issues and requires further investigation.
</Step>
</Steps>

<Warning>
Always test report query changes on your own domain first before modifying a client's query.
</Warning>

## Frequently asked questions

<AccordionGroup>
<Accordion title="What does 'my reports' mean?">
"My reports" refers to reports that are specific to an individual user's settings and configurations.
</Accordion>
<Accordion title="What is a report query?">
A report query is a script that defines what data is retrieved for a report. Unlike "my reports", report queries are shared across all users on a specific domain.
</Accordion>
<Accordion title="What should I do if an email is 'Rejected' in Mailchimp?">
If an email is "Rejected", it typically means it was blocked as spam or blacklisted. Advise the client to contact their IT department.
</Accordion>
<Accordion title="What should I do if an email is 'Delivered' but the user hasn't received it?">
If an email shows as "Delivered" in Mailchimp, proceed with further investigation or contact support.
</Accordion>
</AccordionGroup>
