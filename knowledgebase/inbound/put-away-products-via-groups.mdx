## Putting Away Products via Groups

Product groups let you categorise items (e.g. "Frozen", "Chilled") so that when goods are received, they are automatically routed to the correct storage area. This eliminates manual location selection for routine putaways.

<Steps>
  <Step title="Create or assign product groups">
    If you need new categories like "Frozen" or "Chilled", set up the product groups first. Then either add existing products to the correct groups or assign a group when creating new products.
  </Step>
  <Step title="Create a receipt">
    Create a new receipt for the products you are receiving. Enter the stock quantities for each product — for example, 10 storage units with 10 items each for both Frozen and Chilled products.
  </Step>
  <Step title="Complete receive tasks">
    Complete the receiving tasks as normal. Stock will arrive in your Goods In bay if you have one configured.

    <Tip>A Goods In bay is not mandatory, but it makes the receiving process more organised.</Tip>
  </Step>
  <Step title="Put away stock to grouped locations">
    The **Location To** field automatically shows the correct storage location based on each product's group. If the wrong location appears, use the dropdown to search and select the correct one by typing its code. Complete the putaway task, and the stock will be listed under the correct location for each group.
  </Step>
</Steps>

<Accordion title="Frequently asked questions">
  **What are product groups used for?**

  Product groups sort items into categories and route them to specific storage locations automatically, making organisation simpler.

  **How do I assign a product to a group?**

  Assign the group when creating the product, or edit the product's details later to add it to a group.

  **What if the Location To field shows the wrong location?**

  You can manually select the correct location from the dropdown by entering its code.

  **Can I put away products without using groups?**

  Yes, but using groups means products are routed to their correct locations automatically, which is more efficient.
</Accordion>
